## ドラフト1: PAY.JP営業/サポート宛(Platform利用照会)

### 宛先の見つけ方
- 通常のお問い合わせフォーム: [pay.jp](https://pay.jp) サイト下部「お問い合わせ」、または [help.pay.jp](https://help.pay.jp) 内の問い合わせ導線。
- Platform専用の申請・問い合わせ: [pay.jp/platform](https://pay.jp/platform) ページ内の申込・資料請求フォーム。まずここからPlatform(Payouts型)の利用希望として問い合わせるのが早い。フォームが要件を満たさない場合は上記の一般問い合わせから「PAY.JP Platform(Payouts型)について」と明記して送る。

### 文面

```
件名: PAY.JP Platform(Payouts型)ご利用に関する事前照会

PAY.JP ご担当者様

突然のご連絡失礼いたします。個人事業主として、昆虫(クワガタ・カブトムシ等)の
飼育記録アプリと、飼育者間の個体売買マッチング機能を持つWebサービスを開発して
おります。売買はユーザー間(多くは一般の個人)で直接成立し、運営者である当方は
成約額の5%をシステム利用料として受け取るのみで、代金そのものは一切預かりません。

貴社の「PAY.JP Platform」のうち、テナント(売主)が一般個人でも利用できる
Payouts型の導入を検討しており、契約前に以下をご確認させていただけますでしょうか。

1. 個人事業主が運営するプラットフォームでも、Platform(Payouts型)のプラット
   フォーマー契約は可能でしょうか。
2. Payouts型テナント(売主・一般個人を想定)側に求められる本人確認書類・
   必要情報の要件を教えてください。
3. 通常の加盟店規約にある禁止商材規定「魚類・昆虫のうち適法に販売できるもの
   は販売を認める」という例外は、Platform(Payouts型)のテナント利用規約にも
   同様に適用されますでしょうか。
4. Platformプランの決済手数料率と、発生し得る月額固定費の見積り条件を
   教えてください。
5. 買主側の決済手段について、クレジットカード以外(コンビニ払い等)も
   利用可能でしょうか。
6. 「プラットフォーマーは資金を保持せず、貴社側で買主決済から分配する」
   という構造について、プラットフォーマー側に資金移動業の登録が不要と
   考えてよいか、貴社の一般的なご案内の範囲でご教示いただけますと幸いです
   (最終判断は当方で弁護士にも確認いたします)。

ご多用のところ恐縮ですが、ご回答いただけますと幸いです。何卒よろしく
お願いいたします。
```

## ドラフト3: 弁護士相談用メモ(資金決済法)

### 相談したいこと
PAY.JP Platform(Payouts型)を採用した場合に、運営者(個人事業主)が資金移動業の
登録・前払式支払手段発行者の登録を要しないという理解でよいか、事実関係の整理と
確認したい論点を以下にまとめます。結論は当方では出さず、ご判断を仰ぎたく存じます。

### 事実関係の整理(資金フロー)

```
買主 ──(クレジットカード決済)──> PAY.JP
                                    │
                                    ├─ platform_fee(取引額の5%相当)を控除
                                    │        │
                                    │        └─> 運営者(IHL・個人事業主)へ入金
                                    │
                                    └─ 残額 ──> 売主(テナント・一般個人)へ入金

- 運営者(IHL)は買主のカード決済に一切触れない。代金は買主のカード会社から
  PAY.JP(決済代行事業者)を経由し、PAY.JPが売主口座・運営者口座それぞれへ
  自動的に振り分ける。
- 運営者(IHL)は「取引に platform_fee(5%)を指定してCharge APIを呼び出す」
  役割のみを担い、資金そのものを自社口座で一時的にも保持しない。
- 売主(テナント)は PAY.JP Platform(Payouts型)の利用規約に個別に同意し、
  PAY.JP側のKYC・銀行口座登録を行う(運営者はテナント個別審査を負わない)。
```

### 確認したい論点

1. 上記の資金フロー(運営者が代金を一切保持せず、決済代行事業者であるPAY.JP
   が買主→売主・買主→運営者の双方の資金移動を担う構造)において、運営者
   (IHL)には資金移動業の登録義務が生じないと考えてよいか。
2. 同様に、前払式支払手段発行者としての登録義務も生じないと考えてよいか
   (運営者は残高を発行・管理しない前提)。
3. 「platform_fee」として運営者が受け取る5%相当額は、運営者自身の役務提供
   (マッチング・システム利用)の対価として整理でき、資金移動業の該当性判断
   に影響しないという理解でよいか。
4. 上記の整理は、PAY.JP側の一次情報(利用規約・APIドキュメント)から読み取れる
   範囲での理解であり、PAY.JP自身の法的整理そのものは未確認。契約前にPAY.JP
   の営業・法務担当にも同種の確認を行う予定である旨、申し添えます。
