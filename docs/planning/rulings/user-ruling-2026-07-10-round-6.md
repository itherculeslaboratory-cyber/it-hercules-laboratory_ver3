## 3. 裁定（R6 形式: 結論 + 適用ルール + 理由 +（修正承認時）修正後要求文）

### ① V3-AUT-03 — **修正承認**（適用: R3, R4）

- **理由**: 本人の言葉「ログインをめんどくさくしてほしくない」（R3 欲求ドリブン）。既存 statement（opaque + サーバ側失効管理）は 2026-07-10 以前の裁定であり、本日の裁定が新しい（R4 時代の重み）。技術的裏づけ: ver2 のセッションはインメモリ照合で Workers では方式変更が必至・ステートレスは常駐ストアゼロで不変条項①（10年コスト最小）に最適合・C1 で凍結した actor_id 導出（CL-03 ベクタ）はトークン形式と独立。
- **修正後要求文**: ver3 のセッションは署名付きステートレストークン（HMAC/JWT 系・サーバ側セッションストアなし）を正とする（2026-07-10 第6回裁定で opaque + サーバ側ストア方式から変更）。平文トークンの localStorage 直置きは引き続き禁止し、保持方法は HttpOnly Cookie 等の安全な手段を C2 設計で確定する。失効は短め TTL（30 日目安）+ マジックリンク再ログインで担保し、強制失効が要件化した場合は KV デニーリスト等を後付けする。actor_id 導出（CL-03 凍結・sha256 系）は不変とする。

### ② CL-08 `vector_length` — **承認**（適用: R3, R4）

- **結論**: `vector_length` = **要素数（= embedding_dim = 384）**。ver2 実装（`embedding_builder/run.py`）の実書込値が正、frozen スキーマ description の「1536 バイト長」記述が誤り。
- **理由**: CL-08 の凍結対象は既存 R2 データとの互換。既存 manifest は 384 を持つ（C1 実機照合 — `../c1/REPORT-ver3-phase-c1-2026-07-10.md` 曖昧点1）。
- **実施**: frozen スキーマ description の訂正は C2 で、CL-08 negative TC の同時更新とセットで行う（CI frozen 変更ゲート準拠）。

### ③ CL-03 email 正規化 — **承認**（適用: R3, R4）

- **結論**: ver3 は入口で `strip().lower()` に統一（セッション経路 = 既存 actor_id 主系を正とする）。旧 pii イベントの raw ハッシュは v0 イベントとして封印・遡及変換しない（開発計画 §5.3）。
- **理由**: ver2 内不整合（C1 実機照合 — 曖昧点2）の解消。既存 actor_id の主系（セッション経路）互換を維持する側に寄せる。

### ④ Resend 実鍵 + DNS 投入時期 — **記録のみ（時期委任）**

- ユーザー明言「いつでもいいよ。最適なタイミングで」により、投入タイミングは AI 判断に委任。実行直前の一言報告は維持（R1 の趣旨保存）。
