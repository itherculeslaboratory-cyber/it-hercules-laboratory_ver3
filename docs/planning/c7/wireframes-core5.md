## 1. 観測登録(observation-register) — observation-register ワイヤーフレーム(ユースケース起点・v3 批評反映改稿)

設計の芯: 全痛点は「同じことの繰り返し」に収束する。継承・一括・IoT・テンプレを機械側に押し込み、人間が触るのは「今回変わった差分」だけ。入口は1ボタン、モード選択は出さない、既定は最頻行動=既存個体への追観測。
v3の要点: **令期遷移(脱皮・蛹化・羽化)を第一級イベント化**(F2ステージチップ1タップ+F4行/一括→F5合流→F6間隔を新ステージで再計算)、**次回お世話は保存時に自動登録**(F6は事後表示・オプトアウト)、**死亡は発見する場所すべてで記録可**(F2単体・F4行・クラッチ匹数照合)、**親不明・種不明・登録ゼロの行き止まり全排除**、**手記録棚の毎日を複数棚グリッド1保存に**、**保存後の訂正=新レコード追記の入口**をF6に実体化。

  検索が0件だった時(検索欄の直下に出る):
  ┌────────────────────────────────────────────────────────────┐
  │ 🔍「ムナカタ」該当0件                                          │
  │  ▸ 新規個体として登録(QRラベル発行)→ F1-N                      │
  │  ▸ 新しい棚を作る(一括生成)→ F8                                │
  └────────────────────────────────────────────────────────────┘

── 設計注記
- 候補チップが最上段第一候補: 数百個体を回す人の最頻行動=既存への追観測を0タップ検索で開始。直近スキャン/閲覧/placement近接から機械が並べる。誤新規=幽霊個体の不可逆コストを既定にしない [訂正4][批評2]
- **[まとめて記録]は常設ボタン**: 「来週出張だから今日棚Bを前倒しでマット交換」も履歴ゼロの初回も、機械の閾値判定に関係なくF4へ入れる。100回やる作業の入口を提案に人質に取らない
- **[QRスキャン]の解決先を明記**: 個体ラベル→F2 / 棚ラベル(F8生成のセルQR)→F1-E。最頻導線を遷移マップに固定(minor採用)
- **手記録棚チップを「今日の流れから」に常設候補化**: 3棚を毎日手記録する人(IoT未導入層)の年1000回超の検索タイプ+棚ごと保存を殺す。手記録実績のある棚を束ねて1タップでF1-E複数棚グリッドへ(major解消)。実績が無ければ行ごと消える
- **「お世話予定到来」「割り出し頃合い」はF6/F4が生んだ実体に依存**: 前者はF5保存時の次回お世話自動登録(F6)が当日分をここに浮上させ、後者はF4「ペアリング」/F2「産卵セットへ投入」で発生した産卵セットの投入日+40日を起点に出る。以前はどちらも記録する口が無く宙に浮いていたショートカット [批評R4]
- 「今日の流れから」はショートカット: 中断再開・時限イベント・手記録棚は便利提案であって入口の成立条件ではない。無ければ行が消えるだけで機能は欠けない
- 検索は個体・棚(placement)・親ペアを解決: 棚を選べばF1-E(センサー無し棚のみ)、産卵セットを引けばF3を手動開始。環境観測のintent到達経路をここに一本化(生体フォームに温度欄は恒久に出さない)[訂正2の前提]
- 検索0件の脱出口: 即売で買ってきた成虫1頭・譲受個体は「新規個体として登録」へ。行き止まりを作らない。誤新規はF6の事後マージ候補が安全網 [訂正4]
- 4モード選択UIは存在しない: intentはチップ/常設ボタン/検索解決/カメラの選択から裏で確定 [批評2]
- ライト層(履歴ゼロ): チップ行・流れ行が消え [カメラ] が最大ボタンに。[まとめて記録]は残る(小さく)

遷移: チップ→F2 / [まとめて記録]→F4 / QRスキャン→F2(個体)・F1-E(棚) / 検索→F2・F1-E・F3・F1-N・F8 / カメラ→F1-C / ショートカット→F2再開・F4・F1-E複数棚・F3

── 設計注記
- シャッター直後に追観測候補を既定選択で提示: カメラ最短で撮っても既定は「既存への追加」。新規は明示選択のみ=幽霊個体既定をカメラ枝からも排除 [訂正4][批評2]
- **候補0件の初回は行き止まらせない**: 空の候補リスト+小さな「新規にする」で初回ユーザーを止めない。写真添付済み+ID自動採番+種「不明・あとで確定」既定の1画面で[保存]が成立=正本『写真だけの人は種スキップ可で保存成立』を画面に落とした(major解消)。種は後日候補→確定でリンク [訂正5]
- ライト層の2枚目以降は「前回の子と同じ?[はい]」1タップリンク: 孤児写真12枚・成長タイムラインゼロを構造的に防ぐ
- 候補の並び順は直近スキャン>直近閲覧>撮影placement近接。チップは最大3+「検索」

── 設計注記
- **複数棚グリッドに拡張**: 毎日3棚を記録する人が「検索→1棚保存→戻って再検索」を年1000回繰り返さない。手記録実績のある全センサー無し棚を行に並べ、温湿度2列だけ打って1保存で全棚一括INSERT(major解消)。計測グリッド部品の再利用(F4と同型)
- 到達経路は3本: F1「今日の流れから」チップ(実績棚を束ね済み)/検索で棚ヒット(その棚1行のみ)/棚QRスキャン。単棚時は1行に縮退し[保存]
- 前回手記録を並記: 打ち間違い(58→85等)をその場で気づける
- 生きたセンサーが居る棚は行に出ない: 最新自動値の読み取り表示(鮮度ドット付き)に置換され、重複入力を防ぐ。環境軸の独立を維持し、生体フォームには恒久に温度欄を出さない [訂正2の前提]

遷移: [まとめて保存]→各棚時系列にappend-only INSERTしF1へ戻る(確認画面は不要=棚×2値)

## 2. 検索(obs-navigator) — 検索(観測ナビゲータ) obs-navigator — ASCII ワイヤーフレーム v3(第2批評反映改稿)

正本: `docs/planning/c7/usecase-driven-design.md` §3 observation-search。
目的関数: 頭の中の像(「去年81mm測った越冬中のデカい♂」)から、在庫内の該当個体へ
**取り違えゼロで最短着地 → そのまま計測/出品/血統照合へ束ねて流す**。写真鑑賞は目的でない。

改稿サマリ(M=major・m=minor、番号=第2批評JSONの並び順):
- [M1] バスケット出口後ライフサイクルを定義: 出口成功時に**自動クリア+「流し込みました [バスケットに戻す]」undoスナックバー**。二重流し込み事故と毎朝の余分な[空にする]を両方消す。

### 1. UI深さ=テンプレート明示切替が主・自動判定は補助
- 初期状態=公式ライトテンプレート、ブリーダー向けに公式ヘビーテンプレート(実体=同一 ScreenDef 語彙の畳み方既定値セット×2。二重実装禁止と両立)。

- → 個体詳細 open_questions「深さの既定の持ち方」はこれでクローズ(ユーザープロファイルのテンプレート選択)。

### 2. リマインドの温度感=「追い立てない」(スケジュール管理にしない)
- 予定は「予定」でなく**「目安」**。超過の赤表示・件数バッジで急かさない。「そろそろ」の中立表現のみ。
- individual-detail の triage: due/超過を ■要対応(赤)にしない。■は異常(体重減・環境逸脱)専用。due は中立の「そろそろ」チップへ降格。
- 通知の既定=アプリ内で開いた時に見えるだけ。外部 push は既定OFF(オプトイン)。→ 観測登録 open_questions「通知手段」クローズ。
- 利用頻度の前提を捨てる: 半年1回のマット交換・月1写真でも罪悪感が出ない画面が既定。「今日の流れから」の名称は日次前提を外す方向で次改稿時に再考。
- F6 の自動登録(ゼロタップ)自体は維持 — 文言を「次の目安 登録済み」に。

### 3. 環境データは CSVファースト(常駐 collector/Docker 前提を捨てる)
- 必要な連携は2つだけ: ①ケースQR→棚(タップ選択でも可) ②棚→機器。期間は placement start/end(既に第一級イベント)。
- 環境は観測レコードに焼き込まず時刻×棚 join で参照(ワイヤー既定)→ **CSV 後入れで過去観測の環境も遡及して埋まる**。
- CSV importer を新規採番候補として裁定待ちへ: SwitchBot エクスポート形式(Date,Temperature,RH,DPT,VPD,AbsHumidity[,Light_Value]・1分粒度・実測9〜14万行/ファイル)。取り込み=機器選択→ファイル投入→期間重複は put-if-absent(409)で自動スキップ(再取込も安全・append-only)。常駐 collector は「あれば使う」任意レイヤー。
- F2「自動で記録済み」欄の縮退規定: ライブ機器なしの棚では警告なしで「環境は棚の記録から参照(あとからCSVで埋められます)」1行。鮮度ドットはライブ機器のみの概念。
- 観測登録時に人が持ち込むのは対象データ(計測値・写真)+撮影時刻のみ。
