## 1. 一行で

**個の観測を、10 年後も動く形で、安く、書き換え不能に残す。** 派手さではなく「消えないこと」と「安いこと」を最上位に置いた civic-scale の観測基盤である。

## 2. 中心となる賭け（設計判断）

### 2.1 Truth は append-only、repo の外

一次事実（Truth）は Cloudflare R2 に **INSERT ONLY** で積む。UPDATE / DELETE を持たない。訂正は上書きではなく新しい record / snapshot を追記する。

- **なぜ**: 10 年スパンで「あの時点で何が観測されたか」を再生できることが、便利さより重い。可変ストアは過去を静かに失う。
- **どう担保するか**: 書込は put-if-absent。同一キー再 PUT は 409 で拒否され、既存値は不変。この不変性はアプリ層の作法ではなく **storage 層で強制**する（`packages/truth`）。
- **PII**: 物理削除ではなく不使用フラグで論理無効化する。削除要求にも「積む」で応える。

### 2.2 常駐 DB を SSOT にしない — 派生値は都度再計算

一覧・集計・ランキング・台帳残高といった派生値は**保存しない**。イベント列を prefix scan して呼ぶたびに再計算する（投影純関数 `f`）。

- **なぜ**: 常駐 DB は 10 年間の固定費と運用面を負う。投影は捨てて作り直せる。正本はイベント列だけでよい。
- **代償と上げ代**: スキャンコストは負う。ホット経路が計測上詰まったときにのみ、投影のマテリアライズを**キャッシュとして**足す（正本は増やさない）。

### 2.3 スキーマが唯一の正本、型・モデル・解説は生成物

`schemas/`（JSON Schema draft 2020-12）が全契約の唯一正本。TypeScript 型・Python モデル・人間可読の解説はすべて codegen の生成物であり、手で編集しない。生成の向きは `schemas/ → generated` の一方向のみ。

- **なぜ**: 契約が 1 か所なら、型とドキュメントの乖離が構造的に起きない。乖離は「直すなら正本を直せ」に一本化される。

### 2.4 UI はコードでなくデータ（ScreenDef）

画面は React コードではなく ScreenDef JSON で宣言し、単一の Renderer が実行する。ナビゲーション・権限・データ束縛も宣言的に持つ。

- **なぜ**: 画面を fork・差替・機械検証できる。スナップショット GATE と route 権限整合を、画面そのものに対して機械判定できる。

### 2.5 埋め込みは 384 次元一本、端末オフロード

画像・テキストとも 384 次元に一本化し、端末（WASM / WebGPU / ONNX）で実行できるサイズに固定する。LLM / Vision / FAISS は **既定 OFF**。

- **なぜ**: 10 年コスト最小。重い推論をサーバ常駐にしない。必要な人が必要な瞬間に、自分の鍵（BYOK）で足す。

## 4. AI の位置づけ

AI は主役ではなく**差替可能なプロファイル**として周縁に置く。機能ごとに provider / model / compute_tier を宣言（`ai-profiles/*.json`）し、鍵はサーバに保持しない（BYOK）。ライブ配線は opt-in まで既定 OFF。

- 二層ビュー: 人間向け要約（`human_view`）と機械向け構造（`machine_view`: sections / keypoints / entities / topics / rag_chunk / importance）を分離し、同じ Truth を人と機械の双方が読める形にする。

## 5. 開発プロセスそのものを機械ガード化する

思想を人の善意に頼らず、CI の GATE で強制する。

- **仕様先行（Spec-Driven）**: データモデル・付与条件・ナビ・権限・スレッドを、実装前に spec JSON で明文化する。
- **意図台帳**: なぜその変更をしたか（intent_summary / problem_statement / expected_effect / rejected_alternatives / decision_source）を append-only の意図イベントとして残す。
- **要件凍結ゲート**: 要件本文の変更には裁定参照を必須にする。
- **危険コードの物理ゲート**: 金銭系 API 実行・DNS/ドメイン操作・自己権限変更を含む差分は機械が止める。
- **RTM 閉包**: 全実装対象要件が最低 1 つの TEST 層参照へ閉包しなければ lint が fail する（テスト設計ゲートを免除しない）。

## 6. やらないこと（優先順位）

**消えないこと > 安いこと > 検証できること > 拡張できること > 便利なこと。**

上位が満たされない機能は、どれだけ便利でも採らない。便利さのために不変性や安さを削らない。
