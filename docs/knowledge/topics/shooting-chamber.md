# ヘラクレス標準撮影チャンバー v1

**目的**: 全国の参加者がスマホで同一品質のヘラクレス画像を取得し、色解析・サイズ計測・AI 学習用データセットを構築する。設計思想は**高精度より再現性優先** — 全員が同じ照明・同じ距離・同じ背景・同じ色基準で撮る。

## 撮影ボックスと照明

- 外形 40×40×40cm、内装マットグレー（光沢禁止）、背景はマットグレー / 18% グレー。
- LED × 4 灯（同一型番・**CRI95 以上**・**6500K 固定**）を左前・右前・左後・右後の 45° に配置。
- スマホ 1 台固定、レンズ中心から **35cm 固定**。

## カメラ設定（撮影条件の固定）

RAW（DNG）· ISO 固定 · WB 固定 6500K。**HDR / ナイトモード / AI 補正 / フィルター / フラッシュはすべて禁止** — 自動処理を排して撮影条件を再現可能にするのが狙い。

## 色基準・サイズ基準

- **色**: ColorChecker Nano を固定ホルダーで毎回同じ場所に設置。内壁に白・18% グレー・黒の基準バッチを常設し、全画像に最低 1 組以上写る構造。
- **サイズ**: 底面に 5mm グリッド印刷、四隅に ArUco マーカー配置（サイズ換算・姿勢補正・距離確認に利用）。

## 被写体保持と回転

中央の透明ポリカーボネート棒（径 20〜25mm・高さ約 250mm）下部に誘導ステージを置き、ヘラクレスが自然に登る構造。電動ターンテーブルで**棒ごと回転**し 0°・90°・180°・270° で撮影。虫を直接固定しない低ストレス設計。

## 撮影 5 視点とフロー

視点は上面・正面・背面・左側面・右側面の計 5 視点。フロー: 誘導ステージへ置く → 棒へ登る → アプリが位置確認 → 回転台自動制御 → 5 視点撮影 → RAW 保存 → 自動アップロード。保存形式は `sample_id.dng`（正）+ `sample_id.jpg`（確認用）。

## 色補正パイプラインと特徴量

```
ColorChecker 検出 → 24 パッチ抽出 → 基準値比較 → カラープロファイル生成
                 → RAW 補正 → XYZ 変換 → Lab 変換
```

出力は補正 RGB / XYZ / Lab。A1 特徴量は部位別平均 L*a*b*（頭部・胸角・前胸・上翅）+ 各部位分散 + 色ヒストグラム + DINOv2 / CLIP / ViT 埋め込み。

```
embedding = [DINOv2 (768), ColorHist (96), LabFeatures]
```

## 観測パイプラインとの接続

この標準化撮影は [observation-pipeline](./observation-pipeline.md) の入力品質を担保する層にあたる。撮影チャンバーで色補正済み・視点固定の高品質 RAW を得ることで、下流の DINOv2 埋め込みと類似検索 rerank（色・サイズ成分）が意味を持つ。ただし観測パイプライン側の類似検索 preview は「色補正なし」の raw 条件を明示表示する点と役割が異なる（撮影標準化 = データ生成側、パイプライン = 検索側）。
