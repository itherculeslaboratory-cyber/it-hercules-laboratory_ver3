## 思想

**個の観測を、10 年後も動く形で残す。** 生き物・道具・取引 — 現場で一次記録された事実
（Truth）は、書き換えられず、安く、誰の手からでも再生できなければならない。派手な機能では
なく、消えないことと安いことを最優先に置く。

## 5 つの不変条項（全要件・全設計に優先する）

1. **10 年ランニングコスト最小** — 常駐 DB を SSOT にしない。派生値は都度再計算。
   LLM / Vision / FAISS は既定 OFF。ID・Index は使う瞬間だけ発行する。
2. **フォーク文化** — 正本は GitHub 一本・全ソース単一 clone 公開。改善は Component 単位の
   fork + lineage。少数固定の骨格に fork の拡張を重ねる。
3. **Truth は append-only** — 一次事実は INSERT ONLY。UPDATE / DELETE は禁止。修正は新しい
   record / snapshot として積む。PII は物理削除ではなく不使用フラグで論理無効化する。
4. **人間ゲート文化** — 不可逆・公開・金銭・実鍵・物理治具は人間が裁定する。設計が確定する
   までは実装しない。sandbox の改善は明示的な Promote があるまで本番へ流さない。
5. **検証されないものは納品されない** — 独立批評家（正しさ / 回帰 / 設計整合）を通過することが
   納品条件。機械 GATE の PASS を、人間の「完成宣言」より優先する。

## 構造

- **Truth は repo の外**（append-only ストレージ）。repo が持つのは契約（`schemas/`）と、
  投影を再生成する純粋関数だけ。
- **スキーマが唯一の正本。** 型・モデル・解説は生成物であり、手で編集しない。正本を直せ。
- **秘密は repo に入れない。** 置いてよいのは `.env.example` の型だけ。実鍵の投入は人間ゲート。

## 優先順位（迷ったらこの順）

**消えないこと > 安いこと > 検証できること > 拡張できること > 便利なこと。**

便利さのために消えない性質や安さを削ってはならない。上位が満たされない機能は、どれだけ
便利でも採らない。
